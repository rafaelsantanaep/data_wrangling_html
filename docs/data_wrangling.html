Introduction¶

The data used in this project consists of three data-frames that must be transformed into one dataframe.

Two datasets were provided by Udacity, and the third will be created by gathering data using the tweeter api (tweetpy).

Before combining the datasets, it will be necessary to clean the data, since there are a lot of issues on it.

I have already assessed the datasets to check the data quality and tidiness issues, and I have created a list of all the issues that will be addressed in data cleaning section. After that, I'll combine the datasets and I'll start exploring interesting relationships that exists in this data.

Assessing¶

Issues in the dataframe¶

I have already lookout for the problems in the three datasets. Here are the problems that I have found:

Quality Issues¶

Tweets¶
  1. There are missing data related to the retweets and replies. Those columns are ids, but they're useful to find which rows are retweets. After removing the rows that are retweets, I'll also remove those columns.
  2. Wrong dtypes in the columns timestamp (object), tweet_id (int64).
  3. In the column name, there are some names that are not names like 'a', 'an', etc.
  4. There is an accuracy issue in the columns rating_numerator and rating_denominator. This problem is happening because the ratings are extracted of the body of the tweet. And some of the tweets, has more than one piece that look like a rating in its text. Besides that, the regex compiler created by the Udacity instructor is not capturing ratings that have decimals on it.
  5. The regex compiler used to extract the dog types has room for improvement.
Image Predictions¶
  1. Some of the dogs's breeds are written in lower case, others in upper case and there is a underline in many of them.
Twitter API:¶
  1. One tweet have failed when the api was trying to retrieve them.
  2. There are retweets data that are affecting the actual values of the retweet_count variable
Merged dataframe¶
  1. The number of rows in each dataframe is different. So, if we do a inner join, some information will be lost and if we do a left join, right join or full outer join, there will be missing values in our dataframe. As I don't have access to the code used to create the neural networks to classify the dog's breeds. I will use a inner join to not create bias in our dataset.

Tidiness Issues¶

Tweets¶
  1. There is a url in the text column. Since, we have already an url for the images, I'll drop this part of the column.
  2. The observational unity} of the dataset is the picture, but there are some pictures that has more than one dog. This have some consequences like having more than one dog type per column and, sometimes, two ratings.
  3. The variable ratings are splitted in numerator and denominator.

Define¶

There are four types of dogs inside the vocabulary of the community: Doggo, Pupper, Floofer and Puppo. However, we have only the text of the tweet to identify the type of the dog. The dataset provided by Udacity already have those dogtypes distributed in four different variables.

There are two problems with those variables.
1- I want to group them into one variable only (Tidyness issue)
2- There are some places where there are more than one dog and the code used to extract is not getting those dogtypes, just the first group.
3- The number of missing values is very huge, I'll try to get a better result by doing a programatically (re.compile) and a visual assessment at the same time. (pd.value_counts()). In the visual assessment, I'll create an dictionary to transform the wrong values in the right values.

Define¶

The rating_numerator and rating_denominator has problems with both tidiness issues and quality issues. The variable is untidy because is splitted into two columns and as we are going to do operations with this column in the data analysis section of this project, is better to divide the numerator by the denominator. It has also quality issues because there are columns where the rating captured by the regex is not the right one because there are columns that has more than one string that could represent a rating. Sometimes, there are two ratings because there are two dogs in these column.

In order to solve this problem, I'll create a regex pattern to retrieve all the ratings in each of the rows. If there are more than one ratingm, then I'll visually assess it in order to be accurate.

Define¶

There are some patterns in the way that the WeRateDogs's owner write his tweets. I believe that the person who have extracted tried all the patterns, however, he not restrain it to get only names and, because of that, he get a lot of string that aren't names.

One way to clean was to only replace those words for 'None'. However, I'll try to see if some kind of pattern was not in the scope of the person who have capture this data.

There are some rows that have more than one dog. I want to capture it too.

Patterns¶
  • That / This is dogname
  • named 'dogname'
  • His / Her name is dogname
  • dogname1 and dogname2
  • Their names are dogname1 and dogname2
  • Here is dogname
  • Meet dogname
  • Say hello to dogname

I will create a function that uses re.compile and re.findall to extract the strings and store it in a dictionary. After that, I will collapse all the patterns into one string using the join function. Lastly, I'll use the map function to store it in the dataframe using the ids of each tweet.

pattern = "([A-Z][a-z]*)\sand|Their\snames\sare\s([A-Z][a-z]*)|[H|h]ere\sis\s([A-Z][a-z]*)|[This|That|this|that] is ([A-Z][a-z]*)|named ([A-Z][a-z]*)|Meet\s([A-Z][a-z]*)| [His|Her|his|her] name is ([A-Z][a-z]*)|[s|S]ay\shello\sto\s([A-Z][a-z]*)"

# pattern to get the second dog name (when there is one)
pattern2 = "[A-Z][a-z]*\sand\s([A-Z][a-z]*)"

def extract_and_transform(pattern):
    """
    This function is used to create a regex compiler an return a dictionary
    that can be used to map values in the text variable of our dataframe.
    
    This regex pattern will detect the names, append it to a dict. 
    
    The dictionary will have a bunch of lists in each of the keys. Hence, I'll use the
    join function to combined with a conditional statement to check if there is 
    something in the list. If the list is empty, the it will create an string 'None' 
    and append it to the dictionary.'
    
    ** Arguments:
    
    - pattern: regex pattern to be compiled
    
    """
    names = re.compile(pattern)
    # the strings extracted will be the values and the tweet ids will be the strings
    di = {}
    for i in range(len(tweets)):
        di[tweets['tweet_id'][i]] = names.findall(tweets['text'][i])
        
    # iterating over the dictionary and using a conditional statement
    names = {}
    for i, j in zip(di.keys(), di.values()):
        if len(j) != 0:
            string = j[0]
            names[i] = ''.join(string)
        else:
            string = 'None'
            names[i] = ''.join(string)
    # returning a dictionary
    return names

# using the function to return two dictionaries
names_d1 = extract_and_transform(pattern)
names_d2 = extract_and_transform(pattern2)

# mapping the values
tweets['names_two'] = tweets['tweet_id'].map(names_d1)
tweets['names_three'] = tweets['tweet_id'].map(names_d2)

Most common breeds of dogs¶

Udacity's team had created a CNN model to classify the breeds of the dogs in each picture. This model has yield three guesses with the percentage of confiability of each of them.

As you will see, the first prediction always have higher probability of being right than the second and the third prediction according to the model. To exemplify this, I'll get the average confiability for each of the predictions below.

Popularity Metrics of a tweeter page¶

In Tweeter, there are two main metrics for measuring the engagement with a page.

  • The likes (stored in the favorite_count variable).
  • The retweets (stored in the retweet_count variable).

I believe that those variables have a high correlation, since the more a post is retweeted, the more likes it gain.

But, before investigate it, I wanna look at the distribution of both variables.

The distributions of both variables are pretty similar in its shape, but they are on different scales. Both of them are right skewed, so the means (2645 and 8717) are higher than the medians (1275 and 3690).

There are a lot of outliers in both variables. In the first variable, for example, the maximum value is 83848 and the third quartile limit is just 3410. In order to avoid that in the scatter plot, I'll limit the range of values to the 97.5 percentile.

One episode that is famous among community members is the episode when one fan of the page has criticized the rating system of the plataform. As you can see below:

Conclusions¶

Main takeaways¶

  • The variables favorite_count and retweet_count have a high correlation
  • The correlation between the score and the measures of popularity is almost 0
  • The number of tweets created by month have been decreasing over time
  • The most common breeds predicted by the CNN are the Golden retriever, Labrador Retriever, Pug, Chihuawa and Pembroke
  • Within the dogs that have a dogtype explicitly, the most common are the doggo and the pupper.

Limitations¶

There are a lot of other things that could be explored, for example, the daily activity of the followers of the page, the number of likes and retweets by day or month. However, we don't have the data available to measure these variables.
